# Plan: Make Blog Dynamic and Deployable to Node.js Hosting

## Goal
Convert the blog from static generation to dynamic rendering so content loads without rebuild, and prepare for deployment to Node.js hosting platforms like Hostinger.

## Current State Analysis
- Blog list (`/blog`): Already dynamic - uses client-side fetching from Supabase
- Individual posts (`/blog/[slug]`): Uses static generation with `generateStaticParams`, requires rebuild for new posts
- Content stored in Supabase database
- Admin interface exists for creating/editing posts

## Selected Approach: Option B (Separate Express App)
Create a standalone Node.js Express application for the blog that reads from the same Supabase database as the main Next.js app. Admin functionality remains in the main app, while the blog app provides public-facing dynamic content.

## Key Requirements
- Separate Node.js app using Express and EJS templating
- Shares Supabase database with main app for real-time content updates
- No admin functionality in blog app (admin stays in Next.js app)
- Identical design/CSS/layout as current /blog page
- Dynamic content loading without rebuilds
- Deployable to Hostinger Node.js hosting

## Implementation Steps

### 1. Create New Blog App Structure
- Create new directory `blog-app/` in project root
- Initialize with `npm init -y`
- Install dependencies: express, ejs, @supabase/supabase-js, dotenv
- Set up basic Express server with EJS templating

### 2. Recreate Blog Functionality (Public Only)
- **Routes:**
  - `GET /` - Blog list page (pagination, filtering by tags)
  - `GET /:slug` - Individual blog post page
- No admin routes (admin remains in main Next.js app)

### 3. Database Integration
- Connect to existing Supabase database (same as main app)
- Fetch blog posts dynamically on each request
- Handle image URLs from Supabase storage

### 4. UI Recreation (Match Current Design)
- Extract CSS from `src/app/globals.css` and current blog pages
- Copy hero section, navigation, and layout structure
- Convert React components to EJS templates
- Maintain responsive design and styling
- Include breadcrumb navigation, pagination, and related links

### 5. Configuration and Deployment
- Set up environment variables for Supabase connection
- Configure server for production (PORT, static files)
- Add start script: `node app.js`
- Deploy to Hostinger Node.js hosting

## Dependencies to Install
- express
- ejs
- @supabase/supabase-js
- dotenv

## Data Flow
1. Admin creates/edits posts in main Next.js app (`/admin/blog`)
2. Posts stored in shared Supabase database
3. Blog app reads posts dynamically from Supabase on each request
4. No rebuild needed - content updates in real-time

## Deployment Steps for Hostinger
1. Upload `blog-app/` directory to Hostinger Node.js hosting
2. Run `npm install` on server
3. Configure environment variables (SUPABASE_URL, SUPABASE_ANON_KEY)
4. Set start command to `npm start`
5. Ensure Supabase database connectivity
6. Update any links in main app to point to the blog app URL
